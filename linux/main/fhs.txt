Каталоги могут также содержать другие каталоги. Подкаталогом называется каталог, находящийся внутри
другого каталога. Согласно этому определению все каталоги являются подкаталогами каких-либо
родительских каталогов, за исключением каталога в корне дерева файловой системы, / Этот каталог так
и называется корневым каталогом.


--- FHS стандарт
Первичная иерархия, это корень '/'

/bin                  содержит двоичные файлы, программы нужные для функционирования системы
/boot                 файлы, которые связанные с загрузкой (grub / syslinux)
/etc                  системные конфигурационные файлы системы
/etc/pam.d/           настройка pam для конкретного приложения
/etc/xdg/autostart/   приложения которые запускаются после графической инициализации
/lost+found           нужен для частичного восстановления файловой системы
/srv                  здесь хранятся директории различных сервисов, например ftp, http
/lib                  здесь хранятся lkm, драйвера, модули, библиотеки
/media                точки монтирования съемных носителей
/mnt                  точки монтирования, временное монтирование каких-то файловых систем пользователя
/opt                  необязательное ПО
/proc                 виртуальная файловая система, информация о работе ОС
/sbin                 системные бинарные файлы, запускаются от имени суперпользователя
/tmp                  временное хранилище для временных файлов
/usr                  файлы и программы используемые всеми пользователями
/usr/include          хранятся headers, так же в /usr/lib хранится подобное
/usr/share            хранит файлы для бинарных файлов из /usr/bin, например /usr/share/icons/
/usr/src              хранится source код
/var                  все упоминавшиеся выше каталоги остаются статичными, их содержимое не изменятся,
                      но var изменяется часто, здесь хранятся кэши, базы данных, бекапапы и т.д
/var/log              журналы системы
~/.local/share/Trash  корзина
/run                  хранятся файлы, которые когда-то лежали в /etc
/sys                  монтируется вирт-файловая система sysfs (информация об устройствах)
/sys/class            яркость на ноутбуке /sys/class/backlight/acpi_video0/brightness
                      MAC адрес /sys/class/net/enp1s0/address и другое
/dev                  здесь ядро хранит список всех известных ему устройств, это не файлы, они хранятся в памяти
                      и не все файлы связанные с какими-то устройствами, есть виртуальные устройства по типа /dev/null


--- Устройства
/dev/sda              первый жесткий диск
/dev/sda1 /dev/sda2   разделы жесткого диска
/dev/sdb              вторичный жесткий диск, SSD, USB
/dev/cdrom            cdrom (cd/dvd диски)
или /dev/sr0          cdrom (cd/dvd диски), еще /dev/dvd
/dev/ptmx             псевдотерминал
/dev/pts              псевдотерминал
/dev/tty              терминал
/dev/null             псевдоустройство - пустота, черная дыра
/dev/zero             бесконечные нули
/dev/full             операция записи в этот файл завершается ошибкой "нехватка места"
                      устройство записи переполнено

/dev/{random,urandom} генератор случайных чисел, который выводит шум
                      из драйверов устройств и других источников

/dev/zero             псевдоустройство - источник нулевых байтов
/dev/mem              физическая память компьютера
/dev/kmem             виртуальная память

random гененирует данные путем действий пользователя
urandom без действий


--- Процесс
/proc/PID/cmdline    команда, которая запустила процесс
/proc/PID/environ    окружение процесса - переменные
/proc/PID/maps       адресное пространство процесса
                     к числу отображаемых файлов относятся исполняемый файл процесса,
                     а также загруженные библиотеки
/proc/PID/cwd        ссылка на рабочий каталог процесса
/proc/PID/exe        ссылается на исполняемый файл
/proc/PID/fd         каталог, содержит символические ссылки дескрипторов,
                     которые открыты процессом (fd - file descriptor)

/proc/loadavg        общая загруженность
/proc/cpuinfo        информация CPU
/proc/swaps          подкачка
/proc/self           ссылка на текущий процесс,
                     какой используется $ ls -ld /proc/self/exe
/proc/sys/kernel/    информация о ядре
/proc/version        версия ядра и информация

/proc/sys/kernel/hostname
/proc/sys/kernel/{osrelease,ostype,version}
Утилита uname берет информацию из этих файлов

/proc/meminfo        информация о памяти
/proc/mounts         смонтированные ФС
/proc/loadavg        общая загруженность системы, первые три показателя - это число
                     выполняющихся процессов за последние 1,5,15 минут, 3/397 - число выполняемых задач
                     и pid последнего процесса


--- Журналирование
/var/log/syslog      системный журнал
/var/log/messages

Содержат глобальный системный журнал, от ядра Linux, различных служб,
обнаруженных устройствах, сетевых интерфейсов и т.д.

/var/log/auth.log    авторизация пользователей, удачные и неудачные входы, аутентификация
/var/log/boot.log    информация, которая пишется при загрузке системы
/var/log/cups        все, что связано с печатью и принтерами
/var/log/apt         история apt
/var/log/lastlog     список последних пользователей сессии ($ last)
/var/log/faillog     неудачные попытки входа в систему ($ faillog)
/var/log/kern.log    журналы ядра, пользовательские модули
/var/log/dpkg.log    история dpkg
/var/log/user.log    журналы во время пользовательской сессии
/var/log/daemon.log  журналы различных демонов (сервисов)
~/.xsession-errors   вывод stderr графических приложений X11


--- Разное
/etc/services        TCP протоколы
/etc/protocols       IP протоколы
/etc/ssh/            настройка ssh
/etc/passwd          информация о пользователях (сопоставление имени и id)
                     user_name:x:user_id:group_id:GECOS:home:оболочка пользователя

/etc/shells          доступные оболочки
/etc/group           информация о группах (они нужны для совместного пользования файлов)
                     чтобы посмотреть группы в которых состоит пользователь можно выполнить groups

/etc/shadow          пароль
/etc/shadow-         резервная копия файла /etc/shadow
/etc/gshadow         пароль групп
/etc/hostname        имя хоста
/etc/hosts           DNS соответствие, домену соответствует IP

Например: 127.0.0.1 my-pc pc означает что my-pc домен будет запрашиваться с 127.0.0.1
Можно убедиться пингуя my-pc домен, а pc это alias

Можно запретить вход на ресурс
0.0.0.0   www.example.com
0.0.0.0   example.com
::0       www.example.com
::0       example.com

/etc/resolv.conf      DNS конфигурации

Перечисляет серверы имен, которые используются хостом для разрешения DNS Содержит информацию,
которая позволяет компьютеру преобразовывать буквенно-цифровые доменные имена в цифровые IP адреса,
при использовании DHCP данный файл перезаписывается.

nameserver  IP адрес DNS сервера, который будет преобразовывать имя в адрес
            Это может быть личный DNS сервер, либо DNS сервер интернет провайдера
domain      Доменное имя локального хоста
search      Какие доменные имена искать

/etc/network/interfaces
конфигурация сетевых интерфейсов, можно прописать как статические IP
так и DHCP настройку

/etc/bash.bashrc     общесистемная версия окружения и для root и для всех остальных
/etc/profile         глобальный скрипт инициализации который запускается при входе в систему каждого пользователя
/etc/profile.d/      переменные окружения для всей системы
/etc/login.defs      настройка login пакета (вход в систему для всех пользователей) umask, mail_dir и т.д
.bashrc              окружение пользователя
.bash_profile        читает .bashrc
.profile             если нет .bash_profile

/usr/share/timezone  временные зоны
/etc/localtime       локальное время

/etc/sudoers
команда sudo настраивается с помощью данного файла,
никогда не стоит редактировать этот файл с помощью обычных текстовых редакторов

/etc/fstab
это демон монтирования, который срабатывает при включении и отключении ОС
в этом файле прописанны файловые системы, которые будут интегрированны

/etc/mtab
является символьной ссылкой на /proc/self/mounts в котором хранится таблица смонтированных ФС

/etc/sysctl.conf
параметры ядра, а через sysctl можно управлять этими параметрами
(это не тоже самое, что /proc/cmdline, там скорее опции ядра)
